传统 Behavioral Cloning:
┌─────────────────────────────────────────────────────────────────┐
│ 模型权重 = 直接编码特定任务的策略                                │
│ 需要：大量真实任务演示（难收集，成本高）                          │
│ 限制：每个新任务都需要重新收集和训练                              │
└─────────────────────────────────────────────────────────────────┘

In-Context Imitation Learning (论文方法):
┌─────────────────────────────────────────────────────────────────┐
│ 模型权重 = 编码"理解演示"的元能力                                │
│ 训练：使用"任意但一致"的虚拟轨迹                                  │
│ 推理：通过演示定义任务（无需重新训练）                            │
│                                                                  │
│ 关键洞察：                                                       │
│ 由于模型学习的是元能力，不是特定任务，                            │
│ 所以可以用虚拟的"伪演示"训练                                      │
│                                                                  │
│ 结果：                                                           │
│ • 训练数据可以无限生成（只受计算资源限制）                        │
│ • ShapeNet有55个类别，~51,300个3D模型                           │
│ • 随机组合 → 无限可能                                            │
└─────────────────────────────────────────────────────────────────┘

伪数据生成流程（论文描述）：
────────────────────────────────────────────────────────────────
Step 1: 场景构建
  • 从 ShapeNet 随机选择 2 个物体
  • 随机放置在平面上
  • 初始化 Robotiq 2F-85 夹持器网格

Step 2: 任务定义
  • 随机采样 2-6 个路点
  • 路点位于物体表面或附近
  • 随机指定某些路点改变夹持器状态

Step 3: 轨迹生成
  • 采样夹持器初始位姿
  • 在路点间插值（线性、三次样条、球面插值）
  • 在夹持器状态变化时附着/分离物体
  • 保证间隔均匀：1cm平移，3度旋转

Step 4: 观察渲染
  • 使用 PyRender 渲染深度图
  • 3个模拟深度相机
  • 转换为分割点云
  • 记录夹持器位姿和状态

Step 5: 数据增强
  • 30%轨迹：添加局部扰动（用于恢复行为）
  • 10%数据点：翻转夹持器状态（防止过拟合）

Step 6: 偏向采样 (Biased Sampling)
  • 50%使用完全随机路点
  • 50%偏向常见任务：
    - 抓取（靠近物体+闭合）
    - 放置（远离物体+打开）
    - 推拉（沿物体表面）

关键特性：
  • 不需要物理可行性
  • 不需要运动学可行性
  • 只需"语义一致"
  • 可并行生成
  • 持续替换旧样本（防止过拟合）
