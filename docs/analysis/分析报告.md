## 🔍 发现的问题及解决方案

### 问题 1: 导入错误
**问题描述**: 
- `deploy_sim.py` 和 `sim_utils.py` 使用了错误的模块名 `instant_policy`
- 实际项目安装为 `ip` 包

**解决方案**:
```python
# 修改前
from instant_policy import GraphDiffusion
from utils import *

# 修改后
from ip.models.diffusion import GraphDiffusion
from ip.utils.common_utils import *
```

### 问题 2: 缺少配置文件
**问题描述**: 
- `checkpoints/config.pkl` 文件不存在
- 模型加载需要配置参数

**解决方案**:
- 添加自动配置加载逻辑
- 如果 `config.pkl` 不存在，使用 `ip/configs/base_config.py` 中的默认配置

### 问题 3: 缺少场景编码器权重
**问题描述**: 
- `checkpoints/scene_encoder.pt` 文件不存在
- 这是预训练的场景编码器权重

**解决方案**:
- 添加自动检测逻辑
- 如果文件不存在，自动禁用预训练编码器
- 提示用户可以通过下载完整权重获得更好性能

### 问题 4: 错误的模型API调用
**问题描述**: 
- `sim_utils.py` 调用了不存在的 `model.predict_actions()` 方法

**解决方案**:
- 参考原项目的 `ip/utils/rl_bench_utils.py`
- 使用正确的API: `model.test_step(data, 0)`
- 添加必要的场景编码处理逻辑


### 可选参数
- `--task_name`: 任务名称（如 `plate_out`, `open_box`, `close_box` 等）
- `--num_demos`: 演示数量（推荐 1-5）
- `--num_rollouts`: 评估次数（用于计算成功率）
- `--restrict_rot`: 是否限制旋转（1=是，0=否）
