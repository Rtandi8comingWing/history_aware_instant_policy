# Instant Policy - 伪数据生成策略分析

## 📋 概述

Instant Policy 使用**多层次的数据增强和伪数据生成策略**来提升模型的泛化能力和鲁棒性。这些策略分布在数据预处理、训练加载和模型训练三个阶段。

## 🎯 核心数据生成策略

### 1. **轨迹子采样（Trajectory Subsampling）** 🔄

**位置**: `ip/utils/data_proc.py` - `subsample_traj()` 函数

**策略描述**:
- 从密集的演示轨迹中提取稀疏的关键路点
- 基于**平移距离**和**旋转距离**的阈值进行采样

**关键参数**:
```python
trans_space = 0.01  # 1cm 平移间隔
rot_space = 3       # 3度 旋转间隔
```

**工作原理**:
1. 保留第一个和最后一个点
2. 如果相邻点之间的变化太小（<阈值），跳过该点
3. 如果相邻点之间的变化太大（>阈值），插入中间路点
4. 在夹持器状态变化处添加路点

**代码示例**:
```python
def subsample_traj(traj, grips, pcds=None, trans_space=0.01, rot_space=3):
    subsampled_traj = [traj[0]]
    subsampled_grips = [grips[0]]
    
    i = 1
    while i < len(traj):
        trans_dist = np.linalg.norm(traj[i][:3, 3] - subsampled_traj[-1][:3, 3])
        rot_dist = np.linalg.norm(
            Rot.from_matrix(traj[i][:3, :3] @ subsampled_traj[-1][:3, :3].T).as_rotvec(degrees=True))
        
        if trans_dist < trans_space and rot_dist < rot_space:
            i += 1  # 跳过变化太小的点
        elif trans_dist > trans_space or rot_dist > rot_space:
            # 插入中间点
            ...
        else:
            subsampled_traj.append(traj[i])
            ...
```

**优势**:
- ✅ 减少数据冗余
- ✅ 保留关键动作
- ✅ 避免在抓取物体时产生点云抖动

### 2. **关键路点提取（Waypoint Extraction）** 🎯

**位置**: `ip/utils/data_proc.py` - `extract_waypoints()` 函数

**策略描述**:
- 从完整轨迹中智能提取固定数量的关键路点（默认10个）
- 基于动作重要性和轨迹分段距离进行采样

**工作流程**:
```python
def extract_waypoints(traj, traj_states, num_waypoints=10):
    waypoints = [0, len(traj) - 1]  # 首尾必选
    
    # 1. 添加状态变化点（如夹持器开合）
    for i in range(1, len(traj_states) - 2):
        if traj_states[i] != traj_states[i - 1]:
            waypoints.append(i)
    
    # 2. 添加动作误差小的点（可能是关键停顿点）
    for i in range(1, len(traj_states)):
        err = pose_error(traj[i], traj[i - 1], rot_scale=1)
        if err < 3.5e-3:  # 非常小的变化
            waypoints.append(i)
    
    # 3. 根据剩余需求，在长段中按比例分配额外路点
    wp_segment_dist = [pose_error(...) for each segment]
    extra_wp_segments = ceil(wp_segment_dist * wp_to_add)
    ...
```

**优势**:
- ✅ 自适应采样
- ✅ 保证状态变化点被选中
- ✅ 平衡短段和长段的表示

### 3. **点云子采样（Point Cloud Subsampling）** ☁️

**位置**: `ip/utils/data_proc.py` - `subsample_pcd()` 函数

**策略描述**:
- 统计离群点去除
- 随机采样到固定数量（默认2048点）

**代码实现**:
```python
def subsample_pcd(sample, num_points=2048):
    # 1. 统计离群点去除
    sample_filtered, _ = remove_statistical_outliers(
        sample, nb_neighbors=20, std_ratio=2.0)
    
    # 2. 随机采样到固定数量
    rand_idx = np.random.choice(
        len(sample_filtered), 
        num_points, 
        replace=True if len(sample_filtered) < num_points else False
    )
    return sample_filtered[rand_idx]
```

**优势**:
- ✅ 去除噪声点
- ✅ 统一输入尺寸
- ✅ 保持随机性

### 5. **随机夹持器状态反转（Random Gripper Flip）** 🤏

**位置**: `ip/utils/running_dataset.py` - `RunningDataset.__getitem__()`

**策略描述**:
- 在数据加载时，以一定概率随机翻转夹持器状态（开→闭，闭→开）

**代码实现**:
```python
class RunningDataset(Dataset):
    def __init__(self, ..., rand_g_prob=0.0):
        self.rand_g_prob = rand_g_prob  # 默认0.1（10%概率）
    
    def __getitem__(self, idx):
        data = torch.load(...)
        
        # 随机反转夹持器状态
        if np.random.uniform() < self.rand_g_prob:
            data.current_grip *= -1  # 1 ↔ -1
        
        return data
```

**配置位置**: `ip/configs/base_config.py`
```python
config = {
    'randomize_g_prob': 0.1,  # 10%的概率反转
    ...
}
```

**优势**:
- ✅ 提升对夹持器状态的鲁棒性
- ✅ 防止模型过拟合特定的开合模式
- ✅ 简单但有效的数据增强

### 7. **扩散噪声增强（Diffusion Noise Augmentation）** 🌊

**位置**: `ip/models/diffusion.py` - `add_noise()` 和 `training_step()`

**策略描述**:
- 使用DDIM扩散调度器在动作空间添加噪声
- 这是**核心的伪数据生成机制**

**工作流程**:
```python
def training_step(self, data, batch_idx):
    # 1. 随机采样时间步
    timesteps = torch.randint(0, num_train_timesteps, (batch_size,))
    
    # 2. 添加噪声到真实动作
    noisy_actions, noisy_grips = self.add_noise(
        data.actions, data.actions_grip, timesteps)
    
    # 3. 计算去噪标签
    labels = self.model.get_labels(
        clean_actions=data.actions,
        noisy_actions=noisy_actions,
        ...
    )
    
    # 4. 模型学习去噪
    preds = self.model(data)
    loss = self.loss_f(preds, labels)
```

**噪声添加细节**:
```python
def add_noise(self, actions, grip_actions, timesteps):
    # 转换为6D表示 (平移3D + 旋转角轴3D)
    actions_6d = transforms_to_actions(actions)
    
    # 归一化
    actions_6d = self.normalizer.normalize_actions(actions_6d)
    
    # 添加高斯噪声
    noise = torch.randn(actions_6d.shape)
    noisy_actions = self.noise_scheduler.add_noise(actions_6d, noise, timesteps)
    noisy_actions = torch.clamp(noisy_actions, -1, 1)
    
    # 反归一化并转回4x4矩阵
    noisy_actions = self.normalizer.denormalize_actions(noisy_actions)
    noisy_actions = actions_to_transforms(noisy_actions)
    
    return noisy_actions, noisy_grips
```

**扩散调度器配置**:
```python
self.noise_scheduler = DDIMScheduler(
    num_train_timesteps=100,           # 训练时的时间步数
    beta_schedule='squaredcos_cap_v2', # 噪声调度
    clip_sample=False,
    prediction_type='sample',
)
```

**优势**:
- ✅ 生成无限多样的"伪"噪声动作
- ✅ 模型学习从噪声恢复真实动作
- ✅ 提升对扰动的鲁棒性
- ✅ 这是扩散模型的核心思想

### 8. **随机旋转增强（Random Rotation Augmentation）** 🔄

**位置**: `ip/utils/running_dataset.py`

**策略描述**:
- 对点云应用随机的SO(3)旋转（仅在reconstruction任务中使用）

**代码实现**:
```python
def __getitem__(self, idx):
    data = torch.load(...)
    
    if self.random_rotation and self.rec:
        # 生成随机旋转矩阵
        R = torch.tensor(Rot.random().as_matrix())
        
        # 旋转点云
        data.pos = data.pos @ R.T
        data.queries = data.queries @ R.T
    
    return data
```

**优势**:
- ✅ SE(3)不变性学习
- ✅ 提升对物体朝向的鲁棒性

## 📊 数据处理流程总览

```
原始演示轨迹
    ↓
[1] 轨迹子采样 (subsample_traj)
    → 去除冗余帧
    → 保留关键变化
    ↓
[2] 关键路点提取 (extract_waypoints)
    → 提取固定数量路点（如10个）
    → 用于条件演示
    ↓
[3] 点云子采样 (subsample_pcd)
    → 统计离群点去除
    → 采样到固定数量（2048点）
    ↓
[4] 演示排列组合 (manual)
    → shuffle演示顺序
    → 创建不同组合
    ↓
[5] 保存为.pt文件
    ↓
训练时加载:
    ↓
[6] 随机夹持器反转 (rand_g_prob=0.1)
    ↓
[7] 随机演示数量 (randomise_num_demos)
    ↓
[8] 扩散噪声添加 (diffusion training)
    → 核心伪数据生成
    ↓
模型训练
```

## 💡 关键设计思想

### 1. **多级数据增强**
- 预处理级：轨迹子采样、点云处理
- 加载级：夹持器反转、随机旋转
- 训练级：扩散噪声、随机演示数量

### 2. **物理意义保持**
- 子采样保持轨迹的物理合理性
- 插值确保动作连续性
- 归一化保证数值稳定性

### 3. **扩散核心**
- 噪声添加是最重要的"伪数据"生成
- 从随机噪声逐步去噪到真实动作
- 生成无限多样的训练样本

### 4. **自适应采样**
- 基于距离阈值的智能采样
- 保留关键状态变化点
- 按重要性分配路点

## 🔍 总结

Instant Policy的伪数据生成策略可以概括为：

| 策略 | 类型 | 目的 | 影响 |
|-----|------|------|------|
| 轨迹子采样 | 预处理 | 去冗余 | 高 |
| 关键路点提取 | 预处理 | 智能采样 | 高 |
| 点云子采样 | 预处理 | 统一尺寸 | 中 |
| 演示排列组合 | 预处理 | 增加样本 | 高 |
| 夹持器反转 | 加载时 | 鲁棒性 | 中 |
| 随机演示数 | 训练时 | 泛化性 | 中 |
| **扩散噪声** | **训练时** | **核心伪数据** | **最高** |
| 随机旋转 | 加载时 | SE(3)不变 | 低 |

**核心要点**：
- ✅ **扩散噪声是最重要的伪数据生成机制**
- ✅ 多级增强策略互补
- ✅ 保持物理合理性
- ✅ 可根据任务调整参数
